# v4.6.0 Core Consolidation

This release is deterministically generated from the immutable v4.5.0 production baseline without removing scientific models, Observatory detectors, ASML/Cymer EUV, WebXR, depth occlusion, or sonification.

## Source and release artifacts

- `app-v4.5.0.html` — byte-identical immutable v4.5.0 baseline;
- `_includes/uprs460-core-authority.html` — early version and scene-revision authority;
- `_includes/uprs460-consolidation.html` — late runtime integration, scientific semantics provenance, audit and self-tests;
- `tools/consolidate-v46.mjs` — strict deterministic generator;
- `index.html` — verified self-contained production artifact generated by GitHub Actions.

## P0 changes

- one visible and machine-readable application version;
- actual legacy title owners and both v4.1 integration authorities are centralized under v4.6;
- v4.5 is the sole production XR owner; v4.4 remains a tested fallback API;
- idempotent XR installation with redundant repair timers removed;
- desktop audio polling is destroyed when sound is disabled;
- XR scene cache includes geometry/selection revision rather than object counts only;
- scene revision changes on state/geometry mutations, not on render frames;
- all registered models receive explicit or clearly marked inferred response semantics and detector applicability metadata;
- legacy version tests are explicitly superseded;
- `__UPRS460__.audit()` and six new consolidation self-tests are added.

## Verified status

- 76 models, including ASML/Cymer LPP EUV;
- 71 inline scripts, zero syntax failures;
- Chromium desktop and mobile: 307/307 self-tests;
- zero runtime errors and warnings in tested scenarios;
- verified keyboard parameter control, point selection, stable mobile panels and event-driven idle rendering.

The next phase will physically consolidate the v4.4/v4.5 XR source into one `ImmersiveEngine`, add a preallocated hot path and introduce real-device Quest performance telemetry.
